Anomalies Resolved

Nine issues detected and corrected across audit cycles. No outstanding critical or high-severity items.

01
Cron Scheduler Crash — runningAtMs TypeError

Nightly historian and all cron jobs were non-functional. OpenClaw 4.3 bug: reads job.state.runningAtMs before state initialization on disk-loaded jobs. Added "state": {} to each job in jobs.json.

02
Expired One-Time Cron Job

Stale deal-strategy-reminder-20260405 blocking scheduler. Removed — only nightly historian remains.

03
Auth Profiles Key Desync

auth-profiles.json held a stale Anthropic API key divergent from openclaw.json. Synced both files. Root cause of hot-reload auth failures. Subsequently cleaned up — old Anthropic keys deleted, one clean key remaining.

04
LaunchD KeepAlive Policy

Plist used SuccessfulExit: false — only restarts on crashes. SIGTERM = clean exit = no restart. Changed to KeepAlive: true for unconditional recovery.

05
Docker Path References in Workspace

HEARTBEAT.md and DONT-LET-PHIL-CRASH-OUT.md still pointed to /home/node/.openclaw/. Updated to native paths.

06
Missing Cron State File

No state.json in cron directory. Created with proper initial structure for job execution tracking.

07
Gateway Port Conflict

Manual process holding port 18789 when launchd attempted second instance. Resolved — launchd is now sole process manager.

08
Docker Path Sweep

Global find/sed across all .openclaw/ JSON files. Zero /home/node references remaining.

09
Ollama Boot Race Condition

Ollama failed to start after reboot — launchd plist fired before user session was ready. Fixed by adding ThrottleInterval: 15 to the Homebrew Ollama plist, which enforces a 15-second delay between restart attempts. Also added OLLAMA_KEEP_ALIVE=5m to the plist EnvironmentVariables dict so the setting survives reboots (previously set via launchctl setenv which doesn't persist).

Strategic Considerations

Remaining infrastructure best practices and the three-operator model for maximizing Claude + Jarvis + Mistral.

Remaining Infrastructure Items

!
Eero Plus ($10/mo)

Network-level threat detection for the home network. Gateway binds to 0.0.0.0 — Eero Plus adds an extra defense layer. Recommended in engineering addendum errata.

!
Healthchecks.io Dead-Man Switch

Free tier. Create a check for each service. Hermes pings it on successful health probe cycle. If Hermes itself goes down, the dead-man fires. External visibility into the stack.

!
Pin Ollama Model Versions

Currently running mistral-small3.1:latest. Pin to a specific digest hash so auto-updates don't change inference behavior. Document in REBUILD.md.

!
Log Rotation

Service logs will grow unbounded. Set up newsyslog or simple logrotate for weekly rotation, keep 4 weeks. Applies to Hermes, OpenClaw, n8n, and backup logs.

!
UPS / Power Protection

Mac Studio + 4TB NVMe = two devices that die simultaneously on power loss. A $150 UPS gives clean shutdown time and protects the NVMe from corruption mid-write.
