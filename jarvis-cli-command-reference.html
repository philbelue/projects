Service Control (launchd)

Every Jarvis service runs as a launchd agent. These are your kill switches.

Check Status

Is OpenClaw running? Check 
launchctl print gui/$(id -u)/com.jarvis.openclaw-gateway | head -5
Look for state = running and a fresh PID
Is Hermes running? Check 
launchctl print gui/$(id -u)/ai.hermes.gateway | head -5
List ALL Jarvis services Check 
launchctl print gui/$(id -u) | grep -i -E "jarvis|hermes|openclaw|watchdog"

Restart Services

Restart OpenClaw (Jarvis) Action 
launchctl kickstart -k gui/$(id -u)/com.jarvis.openclaw-gateway
No output = success. Check status after to confirm new PID.
Restart Hermes Action 
launchctl kickstart -k gui/$(id -u)/ai.hermes.gateway
Restart EVERYTHING Nuclear 
launchctl kickstart -k gui/$(id -u)/com.jarvis.openclaw-gateway && \
launchctl kickstart -k gui/$(id -u)/ai.hermes.gateway && \
launchctl kickstart -k gui/$(id -u)/com.jarvis.watchdog-hermes && \
launchctl kickstart -k gui/$(id -u)/com.jarvis.watchdog-openclaw && \
echo "All services restarted"

Stop / Start

Stop OpenClaw Stop 
launchctl kill SIGTERM gui/$(id -u)/com.jarvis.openclaw-gateway
Start OpenClaw (from stopped) Start 
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.jarvis.openclaw-gateway.plist

OpenClaw CLI

Direct access to OpenClaw's command-line interface — bypass Discord entirely.

Check version Check 
openclaw --version
Show all CLI commands Info 
openclaw --help
View current config (formatted) Check 
cat ~/.openclaw/openclaw.json | python3 -m json.tool
Edit config directly Action 
nano ~/.openclaw/openclaw.json
After editing, restart OpenClaw to apply changes.
Execution policy (v2026.4.12+) Info 
openclaw exec-policy show

Active Memory Plugin

Check active-memory config Check 
cat ~/.openclaw/openclaw.json | python3 -c "
import json, sys
cfg = json.load(sys.stdin)
plugins = cfg.get('plugins', {})
mem = plugins.get('active-memory', {})
print(json.dumps(mem, indent=2))
"

Hermes

Heremes#2749 — 100 slash commands, 6 skills, local Mistral via Ollama.

View Hermes config Check 
cat ~/.hermes/config.yaml
Edit Hermes config Action 
nano ~/.hermes/config.yaml
After editing, restart Hermes to apply.
Recent Hermes logs Check 
tail -100 ~/.hermes/logs/hermes.log
Hermes errors only Errors 
tail -500 ~/.hermes/logs/hermes.log | grep -i -E "error|failed|exception"
Model name gotcha: Hermes config uses the raw Ollama model name (mistral-small3.1), NOT the prefixed version (ollama/mistral-small3.1). Run ollama list to get the exact name.

Ollama

Local model server — Mistral Small 3.1, Flash Attention, Q8 KV cache.

List installed models Check 
ollama list
Is Ollama alive? Check 
curl -s http://localhost:11434/api/tags | python3 -m json.tool
Test model directly Action 
ollama run mistral-small3.1 "ping"
Check loaded models & memory Check 
ollama ps
Shows VRAM usage, quantization level, and time since last use.
Pull / update a model Action 
ollama pull mistral-small3.1
Unload model (free VRAM) Action 
ollama stop mistral-small3.1

Logs & Debugging

When something breaks, start here.

OpenClaw logs (last 10 min) Check 
log show --predicate 'senderImagePath CONTAINS "openclaw"' --last 10m
All Jarvis system logs (last 30 min) Check 
log show --predicate 'senderImagePath CONTAINS[cd] "jarvis" OR senderImagePath CONTAINS[cd] "hermes"' --last 30m --style compact
Hermes errors only Errors 
tail -500 ~/.hermes/logs/hermes.log | grep -i -E "error|failed|exception|not found"
Live-follow Hermes logs Stream 
tail -f ~/.hermes/logs/hermes.log
Press Ctrl+C to stop streaming.

Health Checks

Run all of these when you suspect something is off.

Full system health check Check 
echo "=== OPENCLAW ===" && \
launchctl print gui/$(id -u)/com.jarvis.openclaw-gateway 2>&1 | head -3 && \
echo "" && echo "=== HERMES ===" && \
launchctl print gui/$(id -u)/ai.hermes.gateway 2>&1 | head -3 && \
echo "" && echo "=== OLLAMA ===" && \
curl -s http://localhost:11434/api/tags | python3 -c "import json,sys; d=json.load(sys.stdin); [print(f'  {m[\"name\"]}') for m in d.get('models',[])]" 2>&1 && \
echo "" && echo "=== N8N ===" && \
curl -s -o /dev/null -w "  Status: %{http_code}" http://localhost:5678/healthz && \
echo "" && echo "=== WATCHDOGS ===" && \
launchctl print gui/$(id -u)/com.jarvis.watchdog-hermes 2>&1 | head -2 && \
launchctl print gui/$(id -u)/com.jarvis.watchdog-openclaw 2>&1 | head -2 && \
echo "" && echo "=== DONE ==="
One-shot diagnostic. Copy-paste the whole thing.
System memory usage Check 
vm_stat | head -10 && echo "" && ollama ps
Disk space Check 
df -h / | tail -1

Emergency Procedures

If Jarvis is burning API tokens: Check EVERY n8n workflow synthesis node. All should use google/gemini-2.0-flash via the OpenRouter credential. If you see claude-sonnet-4-6 or the Anthropic credential, that's the leak.

Token Burn — Immediate Stop

Kill ALL Jarvis services immediately Emergency 
launchctl kill SIGTERM gui/$(id -u)/com.jarvis.openclaw-gateway && \
launchctl kill SIGTERM gui/$(id -u)/ai.hermes.gateway && \
echo "Both bots killed. Watchdogs may restart them in 120s."
Watchdogs will attempt restart. To fully stop, also kill watchdog plists.

Model Name Mismatch

Verify model name matches Ollama Check 
echo "=== Ollama has ===" && ollama list && \
echo "" && echo "=== Hermes expects ===" && \
grep -i "model" ~/.hermes/config.yaml
Names must match exactly. No ollama/ prefix in Hermes config.

Memory Not Working Cross-Channel

Check active-memory chat types Check 
cat ~/.openclaw/openclaw.json | python3 -c "
import json, sys
cfg = json.load(sys.stdin)
chat_types = cfg.get('plugins',{}).get('active-memory',{}).get('config',{}).get('allowedChatTypes',[])
print(f'allowedChatTypes: {chat_types}')
if 'channel' not in chat_types:
    print('MISSING: channel — cross-channel memory will not work')
"

Upgrade Guide

Rule: Never upgrade during a fire. Stabilize first, upgrade in a calm window.

OpenClaw Upgrade

Upgrade OpenClaw Action 
# 1. Check current version
openclaw --version

# 2. Backup config
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak

# 3. Update (check docs.openclaw.ai/install for method)
# npm: npm update -g openclaw
# brew: brew upgrade openclaw

# 4. Verify
openclaw --version

# 5. Restart
launchctl kickstart -k gui/$(id -u)/com.jarvis.openclaw-gateway
n8n is LOCKED at v2.8.4. Do NOT upgrade n8n. Node 25 breaks isolated-vm. This is a hard rule.
