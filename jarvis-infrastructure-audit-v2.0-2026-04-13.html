    Vault OS Protocols

    Session Bridge Protocol
    Both Jarvis and Claude (Cowork) read three files on session start: working-context.md (active tasks), mistakes.md (error history), open-loops.md (interrupted projects). Write handoff note on session end. Ensures continuity across separate AI sessions.
    Mistakes Protocol
    Append-only error log. Every mistake is logged with: timestamp, description, root cause, fix applied, and concrete rule to prevent recurrence. Read at session start. Currently 10 logged mistakes (M-001 to M-010, including April 13 authorization violation).
    Open Loops Tracker
    Interrupted project log. When a project is paused mid-work, record context, next step, and blocking issues. Prevents lost threads across sessions. Active projects: Vault OS items 8–14, Brain Cron, IXON Sales OS v2.0.
    Brain Cron — Nightly Vault Maintenance
    Scheduled daily at midnight via n8n. Jarvis reads entire vault, synthesizes daily summary, updates SCHEMA.md evolution, prunes old logs, identifies architectural issues. Keeps vault synchronized with operational reality.

    Mutual Watchdog

    Watchdog Architecture
    Two independent watchdog processes monitor bot health:
    com.jarvis.watchdog-hermes: Pings Hermes every 120 seconds. Reports status to Phil via Discord.
    com.jarvis.watchdog-openclaw: Pings Jarvis every 120 seconds. Reports status to Phil via Discord.
    Watchdogs are independent of both bots — cannot fail with them. Each has its own launchd service.

    Alert Conditions

    • Green (120s heartbeat): Bot responsive, stable
    • Yellow (300s+ no response): Possible lag, manual inspection recommended
    • Red (5+ min no response): Bot likely crashed, watchdog triggers emergency Discord notification + launchctl restart attempt

    Symptom
    model 'ollama/mistral-small3.1' not found — Hermes unable to initialize

    Config file had prefixed model name ollama/mistral-small3.1, but Ollama API expects raw model name mistral-small3.1 without provider prefix.
    Fix Applied
    1. Updated ~/.hermes/config.yaml: changed ollama/mistral-small3.1 → mistral-small3.1
    2. Found same error in Jarvis active-memory plugin config — fixed there too
    3. Restarted both services via launchctl
    Diagnosed/Executed
    Claude (Cowork) diagnosed; Jarvis executed fix

    Fire 2: Morning Brief Burning Sonnet Tokens — RESOLVED

    Symptom
    AM Brief (7:30 AM) and Personal AM Brief (7:00 AM) using expensive claude-sonnet-4-6 via Anthropic API instead of free Gemini Flash via OpenRouter.
    Root Cause
    Jarvis changed synthesis node model on April 12 without understanding cost architecture. Intended to improve quality, but violated mandatory cost rule: n8n synthesis MUST use Gemini Flash.
    Fix Applied
    1. Reverted both workflows to google/gemini-2.0-flash via OpenRouter credential
    2. Verified PM Recap, Pipeline Intelligence, Weekly Deal Strategy were already using correct model
    3. Estimated token burn cost: ~$50–100 (unknown exact amount from April 12-13)
    Diagnosed/Executed
    Claude (Cowork) flagged in previous session, confirmed as live fire on April 13; Jarvis executed fix

    Fire 3: Cross-Channel Memory Broken — RESOLVED + EXPANDED

    Symptom
